Authentication

The authentication endpoints allow users to register and log in to the system. These endpoints do not require authentication.

User Registration

Endpoint: /api/auth/register

Method: POST

Description: Registers a new user in the system.

Authentication: Not required

Request Body: JSON representation of EmployeeRequestDTO

Example Request:

POST http://localhost:8080/api/auth/register
Content-Type: application/json

{
    "firstName": "cst31155",
    "lastName": "3115",
    "username": "christian",
    "password": "3115",
    "departmentId": 1,
    "availabilityId": 1,
    "roleId": 1
}

Example Response:

HTTP/1.1 201 Created
Content-Type: application/json

{
    "employeeId": 7,
    "username": "christian",
    "firstName": "cst31155",
    "lastName": "3115",
    "departmentId": 1,
    "availabilityId": 1,
    "roleId": 1
}

User Login

Endpoint: /api/auth/login

Method: POST

Description: Authenticates a user and returns a JWT token.

Authentication: Not required

Request Body: JSON representation of AuthRequestDTO

Example Request:

POST http://localhost:8080/api/auth/login
Content-Type: application/json

{
    "username": "christian",
    "password": "3115"
}

Example Response:

HTTP/1.1 200 OK
Content-Type: application/json

{
    "token": "eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiJjaHJpc3RpYW4iLCJpYXQiOjE3MzI4NDk0NTYsImV4cCI6MTczMjkzNTg1Nn0.pIARO-UOpl7uwX4hH1cQ0vYsuQtVyZrLPGEuwbc8xXQ"
}

Note: Include this token in the Authorization header for authenticated requests:

Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...

Availabilities

GET http://localhost:8080/api/availabilities

        "availabilityId": 2,

Notes

  • All date and time fields should be in the format YYYY-MM-DDTHH:MM:SS.
  • Ensure the Content-Type header is set to application/json for POST and PUT requests.
  • Use tools like Postman to test the API endpoints.
  • For authentication, replace your_jwt_token with the actual token received from the login endpoint.

Authentication Notes

Some endpoints require authentication, while others do not. Refer to the following guidelines:

  • Public Endpoints (No Authentication Required):
    • /api/auth/**
    • /api/availabilities/**
    • /api/departments/**
    • /api/roles/**
    • /docs and /apidocs.html
  • Protected Endpoints (Authentication Required):
    • All other endpoints not listed as public.

For protected endpoints, include the JWT token in the Authorization header:

Authorization: Bearer your_jwt_token
